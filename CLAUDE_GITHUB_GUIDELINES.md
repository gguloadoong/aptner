# Claude Code — GitHub 운영 가이드라인

> Claude Code가 aptner 프로젝트에서 GitHub 작업 전 반드시 확인해야 할 핵심 규칙.

---

## 표준 워크플로우

```bash
# 1. 새 기능/수정 시작
git checkout -b feat/기능명        # 또는 fix/, chore/, style/ 등

# 2. 작업 후 커밋
git add <specific-files>
git commit -m "$(cat <<'EOF'
feat: 변경 내용 요약

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"

# 3. PR 생성
git push -u origin feat/기능명
gh pr create \
  --title "[feat] 변경 내용 요약" \
  --body "$(cat <<'EOF'
## Summary

변경 내용 설명

## Changes

- 변경 1
- 변경 2

## Type of Change

- [x] `feat` — New feature

## Test Plan

- [x] 로컬에서 `npm run dev` 정상 실행 확인
- [x] 변경된 기능 수동 테스트 완료
- [x] 콘솔 에러 없음

## Related Issues

Closes #

---

> Generated by Claude Code (claude-sonnet-4-6)
> Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)" \
  --base main
```

## 핵심 규칙 요약

| 규칙 | 내용 |
|------|------|
| **브랜치** | `main` 직접 푸시 금지 — 항상 feature 브랜치 → PR |
| **서명** | 모든 커밋에 `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>` |
| **PR 서명** | PR body 하단에 `Generated by Claude Code (claude-sonnet-4-6)` 포함 |
| **라벨** | PR 생성 시 label-pr.yml이 자동으로 `ai-generated`, `claude-code` 적용 |
| **CI** | 3개 잡(lint, typecheck, build) 모두 통과해야 머지 가능 |
| **보안** | `.env` 파일 절대 커밋 금지, API 키 하드코딩 금지 |
| **범위** | fe 에이전트 → `client/` 전용, be 에이전트 → `server/` 전용 |

---

## 자동화 흐름

```
feat/xxx 브랜치 생성
  → 커밋 (Co-Authored-By 포함)
  → gh pr create (PR body에 "Generated by Claude Code" 포함)
  → label-pr.yml: ai-generated + claude-code 라벨 자동 적용
  → label-pr.yml: copilot-pull-request-reviewer 리뷰 요청
  → ci.yml: lint → typecheck → build (3개 잡 병렬)
  → CI 통과 → main 머지
  → 브랜치 자동 삭제 (GitHub 설정)
```
