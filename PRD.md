## 1. 서비스 비전 및 목표

### 비전
"아파트 구매와 청약, 더 이상 발품 팔지 않아도 된다."

국내 아파트 시장의 정보 비대칭을 해소하고, 누구나 빠르게 핵심 부동산 정보를 파악할 수 있는 서비스를 만든다. 호갱노노가 가격 조회에 강점이 있다면, Aptner는 가격 + 청약 + 트렌드를 한 화면에서 제공하는 종합 부동산 인텔리전스 플랫폼이다.

### 목표

| 목표 | 설명 |
|------|------|
| 정보 접근성 향상 | 실거래가, 청약 정보를 하나의 앱에서 제공 |
| 의사결정 속도 개선 | 지도 기반 시각화로 복잡한 데이터를 즉시 파악 |
| 청약 기회 포착 | 청약 일정 캘린더와 알림으로 기회 미스 방지 |
| 핫한 단지 발견 | 조회수/거래량 기반 랭킹으로 시장 관심 지역 노출 |

### 핵심 차별점
- 호갱노노: 실거래가 중심, 지도 UI 강점
- 아실: 데이터 분석 깊이 강점
- **Aptner: 청약 + 실거래가 + 트렌드 랭킹을 모바일 퍼스트로 통합**

## 2. 타겟 사용자 페르소나

### 페르소나 1 - 청약 준비생 "김준혁" (33세, 직장인)

- **상황**: 결혼 2년차, 전세 거주 중. 내집마련을 위해 청약을 준비하고 있지만 어떤 단지가 좋은지, 언제 청약이 열리는지 정보가 흩어져 있어 놓치는 경우가 많다.
- **고통**: 청약홈 앱은 UI가 불편하고, 청약 일정을 따로 엑셀로 관리해야 한다.
- **원하는 것**: 내 주거지 근처 청약 단지를 지도에서 한눈에 보고, 분양가 대비 주변 시세를 즉시 비교하고 싶다.
- **행동 패턴**: 출퇴근 지하철에서 스마트폰으로 정보 탐색. 주 3-4회 앱 사용 예상.

### 페르소나 2 - 갈아타기 실수요자 "박지은" (39세, 맞벌이 주부)

- **상황**: 현재 30평대 아파트 거주 중. 자녀 교육 환경을 위해 강남3구 또는 목동으로 이사를 고려하고 있다. 예산 7-10억.
- **고통**: 원하는 지역의 가격 흐름을 알고 싶은데 매물 앱(네이버 부동산)에는 호가만 있고 실제 거래된 가격 추이를 파악하기 어렵다.
- **원하는 것**: 특정 단지의 3년간 실거래가 차트를 보고, 지금이 고점인지 저점인지 판단하고 싶다.
- **행동 패턴**: 저녁 시간대 태블릿/PC로 깊이 있는 조사. 주 1-2회 집중 사용.

### 페르소나 3 - 투자 목적 "이재훈" (45세, 자영업자)

- **상황**: 부동산 갭투자 경험 있음. 현재 시장에서 저평가된 지역 또는 거래량이 갑자기 늘어난 단지를 빠르게 포착하고 싶다.
- **고통**: 네이버 부동산, 국토부 앱, 뉴스를 각각 확인해야 해서 시간이 너무 걸린다. 거래량 급등 단지를 남들보다 먼저 알기 어렵다.
- **원하는 것**: 전국/지역별 핫한 아파트 랭킹과 거래량 급등 알림을 받고 싶다.
- **행동 패턴**: 아침 출근 전 빠르게 확인. 알림 의존도 높음. 일 1-2회 빠른 스캔.

## 4. 각 기능별 사용자 스토리 및 수용 기준

### 4-1. 지도 기반 아파트 가격 조회 (P0)

**사용자 스토리**
> 나는 30-40대 아파트 구매 준비자로서,
> 지도 위에서 단지별 최근 거래가를 한눈에 보고 싶다.
> 왜냐하면 원하는 지역의 가격 수준을 빠르게 파악하고 비교하기 위해서이다.

**수용 기준 (Acceptance Criteria)**
- [ ] 카카오맵이 기본 지도로 렌더링된다.
- [ ] 지도 이동/줌인 시 현재 뷰포트 내 아파트 단지 마커가 자동으로 로드된다.
- [ ] 각 마커에는 단지명과 최근 거래가(억 단위 축약)가 표시된다.
- [ ] 마커 클릭 시 단지 요약 카드(단지명, 최근 거래가, 면적)가 바텀시트로 표시된다.
- [ ] 바텀시트에서 "상세보기" 클릭 시 단지 상세 페이지로 이동한다.
- [ ] 줌 레벨 13 이하에서는 마커 클러스터링으로 처리한다.
- [ ] 지도 로드 시간 3초 이내.

### 4-2. 아파트 실거래가 차트 (P0)

**사용자 스토리**
> 나는 갈아타기를 고려하는 실수요자로서,
> 특정 단지의 과거 2년간 실거래가 추이를 차트로 보고 싶다.
> 왜냐하면 지금이 고점인지 저점인지 판단하기 위해서이다.

**수용 기준**
- [ ] 단지 상세 페이지에 면적별 탭 선택 후 Recharts 라인 차트가 표시된다.
- [ ] X축: 거래 일자, Y축: 거래가(만원 단위)
- [ ] 데이터 범위: 최근 24개월 (기본값), 6개월/12개월/24개월 토글 제공.
- [ ] 차트 데이터 포인트 hover 시 거래일, 거래가, 층수 툴팁 표시.
- [ ] 거래가 없는 기간은 점선으로 처리.
- [ ] 데이터 로딩 중 스켈레톤 UI 표시.

### 4-3. 청약 정보 목록 (P0)

**사용자 스토리**
> 나는 청약 준비 중인 33세 직장인으로서,
> 현재 진행 중이거나 곧 시작될 청약 목록을 보고 싶다.
> 왜냐하면 기회를 놓치지 않기 위해서이다.

**수용 기준**
- [ ] 청약 목록은 "진행 중", "청약 예정", "발표/당첨" 탭으로 구분된다.
- [ ] 각 항목에는 단지명, 위치(시군구), 분양가(시작가), 접수 마감일 D-day, 타입 수가 표시된다.
- [ ] 지역 필터(시도 단위) 제공.
- [ ] 항목 클릭 시 청약 상세 페이지로 이동.
- [ ] 목록은 마감일 기준 오름차순 정렬(기본).
- [ ] 빈 상태(청약 없음)에서는 안내 메시지 표시.

### 4-4. 아파트 단지 상세 페이지 (P0)

**사용자 스토리**
> 나는 특정 단지에 관심이 생긴 사용자로서,
> 해당 단지의 모든 핵심 정보를 한 페이지에서 보고 싶다.

**수용 기준**
- [ ] 단지 기본 정보 섹션: 단지명, 총 세대수, 건축년도, 건설사, 주소.
- [ ] 면적 탭: 전용 면적별(59m², 84m², 109m² 등) 최근 거래가 및 차트.
- [ ] 최근 거래 내역 테이블: 최근 20건, 거래일/층수/면적/거래가 컬럼.
- [ ] 단지 위치 소형 지도 표시.
- [ ] 카카오맵 "길찾기" 외부 링크.

### 4-5. 지역 검색 (P0)

**수용 기준**
- [ ] 검색창에 시/구/동 입력 시 자동완성 드롭다운 표시 (최대 10개).
- [ ] 단지명 직접 검색 가능.
- [ ] 검색 결과 선택 시 해당 지역/단지로 지도 이동.
- [ ] 최근 검색어 5개 저장 (로컬스토리지).

### 4-6. 지역별 핫한 아파트 랭킹 (P1)

**사용자 스토리**
> 나는 투자 목적의 사용자로서,
> 이번 주 가장 관심받는 아파트 단지 랭킹을 보고 싶다.
> 왜냐하면 시장의 관심이 어디로 쏠리는지 빠르게 파악하기 위해서이다.

**수용 기준**
- [ ] 홈 화면 또는 랭킹 탭에 주간 TOP 20 리스트 표시.
- [ ] 랭킹 기준: 조회수(70%) + 최근 1주 거래량(30%) 가중 점수 (가정: 서버에서 집계).
- [ ] 각 항목에 순위 변동 표시 (▲2, ▼1, NEW).
- [ ] 시/도 단위 지역 필터 제공.
- [ ] 매주 월요일 0시 갱신.

### 4-7. 가격 변동 트렌드 표시 (P1)

**수용 기준**
- [ ] 지도 마커 색상으로 가격 추세 표현: 상승(빨강), 하락(파랑), 보합(회색).
- [ ] 기준: 직전 거래 대비 최근 거래 가격 변동률.
- [ ] 단지 카드에 변동률 텍스트 표시 (예: +3.2%, -1.5%).
- [ ] 트렌드 기준 기간 선택: 1개월/3개월/6개월.

## 5. 필요한 외부 API 목록

| API 이름 | 제공처 | 용도 | 비고 |
|----------|--------|------|------|
| 아파트 실거래가 API | 국토교통부 / 공공데이터포털 (data.go.kr) | 단지별 실거래가 조회 | 인증키 필요, 월 1,000회 무료 |
| 청약 공고 API | LH 청약홈 (apply.lh.or.kr) | 청약 공고 목록, 상세 정보 | 공개 API, 인증키 필요 |
| 카카오맵 API | Kakao Developers | 지도 렌더링, 마커, 검색 | 일 30만 호출 무료 |
| 카카오 로컬 API | Kakao Developers | 주소 검색, 좌표 변환 | 카카오맵과 동일 키 |
| 법정동 코드 API | 행정안전부 / 공공데이터포털 | 지역 코드 매핑 | 정적 데이터로 관리 가능 |
| 건물 정보 API | 국토교통부 건축물대장 | 건축년도, 세대수 등 건물 정보 | 국토부 공공데이터 |

### API 제약사항 및 대응 전략
- 국토부 실거래가 API는 월별 단위로 조회 (연간 조회 시 12회 호출 필요). 서버 캐싱으로 대응.
- 청약홈 API 응답 속도 불안정. 서버에서 주기적으로 수집 후 DB 캐싱.
- 카카오맵은 프론트에서 직접 호출. API 키는 도메인 제한으로 보호.

## 6. 주요 화면 목록

### Screen 1 - 홈 / 지도 탐색 화면 (메인)
- **경로**: `/`
- **설명**: 카카오맵이 화면 전체를 차지하며, 뷰포트 내 아파트 단지 거래가 마커 표시.
- **상단 컴포넌트**: 검색창, 평형 필터 칩 (25평/33평/50평/전체).
- **하단 컴포넌트**: 핫한 단지 랭킹 TOP 5 수평 스크롤 카드.
- **플로팅**: 현재 위치 버튼.

### Screen 2 - 아파트 단지 상세 화면
- **경로**: `/apt/:complexId`
- **설명**: 단지의 모든 핵심 정보를 수직 스크롤로 제공.
- **섹션 순서**: 단지 기본정보 → 가격 요약 (최저/최고/평균) → 면적 탭 + 실거래가 차트 → 최근 거래 내역 → 단지 위치 지도.

### Screen 3 - 청약 정보 화면
- **경로**: `/subscription`
- **설명**: 전국 청약 공고 목록. 탭(진행 중/예정/완료)과 지역 필터.
- **각 카드**: 단지명, 위치, 분양가 시작, 마감 D-day, 청약 유형(일반/특별).

### Screen 4 - 청약 상세 화면
- **경로**: `/subscription/:id`
- **설명**: 특정 청약 공고의 상세 정보.
- **포함 정보**: 분양가 상한표, 공급 세대수, 청약 일정 타임라인, 위치 지도, 분양가 vs 주변 실거래가 비교.

### Screen 5 - 랭킹 화면
- **경로**: `/ranking`
- **설명**: 주간 핫한 아파트 TOP 20.
- **필터**: 전국/시도 선택, 정렬 기준(조회수/거래량/가격상승률).

## 8. MVP 범위 정의

### MVP 포함 (P0 전체)
1. 지도 기반 아파트 거래가 마커 조회
2. 아파트 단지 상세 페이지 (기본 정보 + 실거래가 차트)
3. 청약 정보 목록 (진행 중 / 예정 탭)
4. 지역/단지 검색

### MVP 제외 (이유와 함께)

| 기능 | 제외 이유 |
|------|-----------|
| 푸시 알림 | 백엔드 인프라(FCM) 필요, MVP 속도에 우선순위 낮음 |
| 관심 단지 찜하기 | 회원가입/인증 시스템 필요. 로그인 없는 MVP 우선 |
| 청약 경쟁률 히스토리 | 데이터 수집 기간 필요, 초기에 데이터 없음 |
| 학군 정보 | 별도 API 연동 필요, 핵심 가치와 직결성 낮음 |
| AI 시세 예측 | 충분한 데이터 축적 후 검토 |
| 랭킹 화면 | MVP에서는 홈 화면 하단 TOP 5 카드로 축소 제공 |

### MVP 개발 우선순위

```
Sprint 1 (2주): 지도 + 마커 + 기본 단지 정보
Sprint 2 (2주): 실거래가 차트 + 단지 상세 완성
Sprint 3 (2주): 청약 목록 + 검색
Sprint 4 (1주): 통합 QA + 성능 최적화 + 배포
```

### 가정 사항
- 회원가입/로그인은 MVP에 포함하지 않음. 모든 기능을 비회원으로 사용 가능.
- 초기 데이터는 수도권(서울/경기/인천) 중심으로 우선 제공.
- 서버는 Node.js Express API 프록시로 공공 API 키를 보호하고 캐싱.
- 국토부 실거래가 API의 응답 지연을 고려해 서버에서 일 1회 배치로 주요 단지 데이터 갱신.
